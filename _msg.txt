feat: site search (Pagefind) + register fuzzy search (Fuse.js)

Pagefind — static-site search for marketing pages
- pagefind added to devDependencies; eleventy.after hook builds the
  index against the written _site/**/*.html, outputs to _site/pagefind/
- Marketing pages mark their <main> with data-pagefind-body via the
  base layout so the indexer skips /app/* automatically (the attribute
  is omitted under the app tree)
- New /partials/search-modal.njk with a lazy-loaded UI: Pagefind's
  pagefind.js only loads on first open
- Header button + Ctrl/Cmd+K open the modal from anywhere
- Search modal works on marketing pages WITHOUT Alpine (vanilla JS in
  main.js); main.js is the only script the marketing pages need

Fuse.js — fuzzy register search
- 23 KB vendored bundle, loaded only in the app layout
- Register search input now matches via Fuse when available:
  payee (weight 3), category (weight 1.5), memo (1), amount (0.5)
- Verified end-to-end: "wf" matches Whole Foods, "rentl" (typo)
  matches Rent / Landlord, "groceries" matches by category
- Falls back to the previous substring filter if Fuse failed to load

Header refactor
- Hamburger menu + theme toggle moved to vanilla JS in main.js so
  the marketing layout doesn't have to ship Alpine just to flip the
  menu (the app layout still uses Alpine for the rest of the app)
- main.js now exposes: toggleTheme, openSearch, closeSearch,
  toggleSiteMenu on window.ProjectBudget
- Search modal uses [hidden] attribute with a [hidden] !important
  guard to keep CSS from leaking display:flex over the attribute
