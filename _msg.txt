feat: Dexie/IndexedDB persistence + budget alignment fix

Storage: add Dexie.js as a second persistence backend that mirrors
every localStorage write. The live profile, snapshots, daily backups,
and active-profile id all flow to both stores. Reads stay sync from
localStorage to keep the reactive store sync; Dexie is the durable
backup that survives a localStorage wipe.

Versioning is split into two independent surfaces so upgrades don't
break users:
- Dexie schema version (db.version(N).stores(...).upgrade()) governs
  table shapes + indexes in IndexedDB.
- Profile schema version (SCHEMA_VERSION + MIGRATIONS in schema.js)
  governs the bundle shape. A single Dexie table can hold profiles at
  v1, v2, v3 simultaneously; the loader normalizes on read.

Boot flow:
1. _bootDexie opens the DB; sets storageBackend = "mirrored" or stays
   "localStorage" if Dexie is unavailable (private browsing, very old
   browser).
2. One-time migrateLocalStorageIfNeeded copies any existing
   localStorage keys (profile, snapshot, backup, active) into Dexie.
   Idempotent via a meta flag.
3. Reverse-restore: if localStorage is empty but Dexie has profiles
   (user wiped site data, IndexedDB persisted), copy them back so the
   sync-reading store sees them. Toast notifies.
4. Normal in-memory boot.

Mirror writes use a JSON round-trip unwrap() helper. Alpine's reactive
Proxy can't be passed to IndexedDB.put() — structured-clone throws
DataCloneError on the Proxy traps. Strip it first.

Storage backend shown on /app/settings/ Data card.
New docs page: /docs/profiles-backups/how-storage-works/.

Budget table alignment (user-reported via screenshot):
- Editable amount inputs were inset 0.5rem from the cell edge while
  the read-only group totals sat flush — visible misalignment.
- Input padding: 0.3rem 0 (was 0.5rem horizontal). Focus state pulls
  the padding back in so the visible border doesn't crowd the digits.
- Reformat input value with Intl.NumberFormat thousands grouping
  ("1,450.00" not "1450.00") to match the group totals' formatting.

- Intl.NumberFormat/DateTimeFormat: already using.
