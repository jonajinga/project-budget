fix: load PapaParse + reactive list updates

PapaParse was vendored to disk and import-csv.js used window.Papa when
available, but no layout actually loaded the script tag — same class of
bug as the missing D3 fix earlier. CSV imports were silently going
through the small fallback parser. Add it to layouts/app.njk alongside
D3/Popper/Tippy.

Manual-snapshot UI required a page refresh to show new entries because
listSnapshots() reads localStorage directly and Alpine's fine-grained
reactivity only tracks reads of reactive store props. Add a
_listVersion counter on the store; listSnapshots() and listBackups()
"void this._listVersion" to register a dependency; takeSnapshot,
deleteSnapshot, restoreSnapshot, restoreBackup, and _load all call
_bumpLists() to invalidate. Verified end-to-end in headless Chromium:
adding a snapshot now causes the table to grow without reload.
